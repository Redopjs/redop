Redop is a Bun-first TypeScript framework for building MCP servers without hand-writing the transport, schema, and lifecycle plumbing yourself.

Use it when you want to:

- define tools with strong TypeScript inference
- validate input with Zod, Standard Schema, or JSON Schema
- add middleware, hooks, and reusable plugins
- expose resources and prompts alongside tools
- ship over HTTP or stdio with one framework API

## Why Redop

Most MCP servers start the same way: wire JSON-RPC, parse tool input, manage request state, add auth, then bolt on logging and deployment details later. Redop turns those into framework primitives.

- `tool(...)` registers typed tools
- `derive(...)` adds reusable request context
- `resource(...)` and `prompt(...)` expose the rest of the MCP surface
- middleware and hooks shape execution flow
- `definePlugin(...)` lets you package reusable server behavior

## What the API feels like

```ts
import { Redop } from "@useagents/redop";
import { z } from "zod";

new Redop({
  name: "docs-server",
  title: "Docs Server",
  version: "0.1.0",
  description: "Search docs and return answers.",
})
  .tool("search", {
    description: "Search docs",
    input: z.object({
      query: z.string().min(1),
    }),
    handler: ({ input }) => {
      return { query: input.query, results: [] };
    },
  })
  .listen(3000);
```

## What you get

- typed handler input
- request-aware middleware
- global and tool-local hooks
- built-in plugins for logging and authentication
- HTTP and stdio transports
- resources and prompts when your server needs more than tools
