title: Your First Server
description: Build and run the smallest useful Redop server before you add schemas, middleware, or auth.
---

This is the fastest way to get a working Redop server running end to end.

## Copy-paste example

```ts
import { Redop } from "@useagents/redop";

new Redop({
  name: "hello-redop",
  version: "0.1.0",
})
  .tool("ping", {
    description: "Health check",
    handler: () => ({ pong: true, ts: Date.now() }),
  })
  .listen(3000);
```

## What this does

- `new Redop(...)` defines the server identity returned during initialize
- `.tool(...)` registers one MCP tool
- the handler returns plain serializable data
- `.listen(3000)` starts the HTTP transport on port `3000`

## Run it

```sh
bun run src/index.ts
```

Once it starts, your MCP endpoint will be available at `http://localhost:3000/mcp`.

## Why start this small?

Start with one end-to-end tool first. Once the transport works and your client can connect, add schemas, middleware, hooks, resources, prompts, or auth one piece at a time.
